3. CI Architecture (Global)
================================================

CI in this repository is divided into two distinct categories:

------------------------------------------------
3.1 Phase-Specific CI (Governance)
------------------------------------------------

Each Phase owns its CI governance layer:

- Phase 4 CI → Phase_4_Personalization/ci/
- Phase 4.5 CI → Phase_4.5_Localization/ci/
- Phase 7 CI → Phase_7_Games_Recommendations/ci/phase7/

Properties:
- Understand Phase semantics
- Constrain Phase runtime behavior
- CI-only (non-runtime)
- Authoritative for that Phase only

Phase CI MUST NOT be placed at repo root.

------------------------------------------------
3.2 Repo-Level CI (Observability & Alerting)
------------------------------------------------

Repo-level CI contains ONLY Phase-agnostic CI infrastructure.

Current responsibilities:
- Consume CI SUMMARY (log-level contract)
- Aggregate CI health across Phases
- Produce CI observability artifacts
- Enforce alerting thresholds on CI signals

Repo-level CI:
- Does NOT import Phase runtime
- Does NOT evaluate gameplay, localization, or ranking semantics
- Does NOT gate runtime execution

================================================
5. Forbidden Routing (Hard Rules)
================================================

The following are NOT allowed:

- UI / Client → Phase 1–5 (direct)
- UI / Client → Phase 7 (direct)
- Any Phase bypassing Phase 6
- Runtime invocation of CI-only logic
- Phase CI importing repo-level CI observability
- Repo-level CI invoking Phase runtime
